# 招聘网站系统 - 项目总结

## 项目概述

这是一个功能完整的招聘网站系统，包含面向外部求职者的简历投递界面和面向内部HR的管理后台。

## 核心功能

### 1. 外部投递系统 (frontend/index.html)

**职位展示**
- 美观的卡片式职位列表
- 职位详情弹窗展示
- 实时从后端API获取数据

**简历提交（两种方式）**

**方式一：上传简历**
- 支持PDF、DOC、DOCX格式
- 文件大小限制16MB
- 自动保存到服务器

**方式二：在线填写**
- 动态表单，可添加多条教育/工作经历
- 支持添加多项技能
- 自我介绍文本框
- **自动生成Word简历功能** - 系统会根据填写内容生成标准格式的Word文档

### 2. 管理后台系统 (frontend/admin.html)

**用户认证**
- JWT Token认证机制
- 安全的登录系统
- Token自动过期管理

**数据统计看板**
- 总申请数统计
- 各状态简历数量（待处理、审核中、已面试、已录用、已拒绝）
- 最新申请列表

**职位管理**
- 发布新职位
- 编辑现有职位
- 删除职位
- 职位状态管理（激活/关闭/草稿）
- 查看每个职位的申请数量

**简历管理**
- 简历列表展示
- 按状态筛选
- 按职位筛选
- 查看简历详细信息
- 更新简历状态
- 添加HR备注
- 下载简历文件

## 技术实现

### 后端技术栈

**Flask Web框架**
- 轻量级、灵活的Python Web框架
- RESTful API设计
- Blueprint模块化架构

**SQLAlchemy ORM**
- 数据库操作抽象层
- 支持多种数据库（当前使用SQLite）
- 方便的数据模型定义

**JWT认证**
- 无状态的Token认证
- 安全的用户会话管理
- 24小时Token有效期

**python-docx**
- Word文档生成库
- 支持格式化文本、表格、样式
- 自动简历生成功能的核心

### 前端技术

**原生HTML/CSS/JavaScript**
- 无框架依赖，易于部署
- 响应式设计，支持移动端
- 现代化的UI设计

**Fetch API**
- 异步数据请求
- 与后端RESTful API通信
- 错误处理和用户反馈

### 数据模型

**User（用户）**
- 管理员账号管理
- 密码哈希存储
- 角色权限控制

**Job（职位）**
- 完整的职位信息
- 与创建者关联
- 状态管理

**Application（简历申请）**
- 申请人基本信息
- 详细经历（JSON存储）
- 文件路径记录
- 状态流转
- HR备注

## 核心亮点

### 1. Word简历自动生成

位置: `backend/utils/resume_generator.py`

**功能特点:**
- 美观的简历模板
- 自动格式化
- 包含所有简历要素：
  - 个人信息（居中显示）
  - 自我介绍
  - 教育经历（包含学校、专业、学历、时间）
  - 工作经历（包含公司、职位、时间、描述）
  - 专业技能（包含技能名称和熟练程度）

**技术实现:**
- 使用python-docx库
- 自定义字体、颜色、对齐方式
- 结构化的JSON数据解析
- 自动生成文件名（包含姓名和时间戳）

### 2. 灵活的简历提交方式

**上传模式:**
- 适合已有简历的求职者
- 支持常见文档格式
- 文件类型验证

**在线填写模式:**
- 适合没有简历的求职者
- 动态添加/删除经历
- 可选生成Word文档
- 数据结构化存储

### 3. 完整的管理流程

**状态流转:**
```
待处理 → 审核中 → 已面试 → 已录用/已拒绝
```

**每个状态都可以:**
- 添加HR备注
- 下载简历文件
- 查看完整信息

### 4. 安全性设计

**密码安全:**
- Werkzeug密码哈希
- 永不明文存储

**文件安全:**
- 文件类型白名单
- 文件大小限制
- 安全的文件名处理

**API安全:**
- JWT Token认证
- 公开/私有接口分离
- CORS跨域配置

## 部署说明

### 开发环境

使用提供的启动脚本：
- Windows: `start.bat`
- Linux/Mac: `start.sh`

### 生产环境建议

1. **使用生产级Web服务器**
   - Gunicorn (Linux)
   - uWSGI
   - 反向代理: Nginx

2. **数据库升级**
   - PostgreSQL
   - MySQL
   - 修改 `app.py` 中的数据库URI

3. **安全加固**
   - 修改SECRET_KEY
   - 启用HTTPS
   - 配置防火墙
   - 限制CORS来源

4. **性能优化**
   - 启用数据库索引
   - 添加缓存（Redis）
   - CDN加速静态资源
