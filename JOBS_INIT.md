# 德元升中医馆职位初始化指南

本文档说明如何初始化德元升中医馆的职位数据。

## 职位分类

系统按以下三个分类组织职位，从高到低显示：

### 1. 管理类
- 馆长（1名）
- 副馆长（2名）

### 2. 技术人员类
- 中医执业医师（3名）
- 健康顾问/售后人员（20名）
- 网络/社区运营专员（3名）
- 综合专员（前台/收银/客服/导诊）（4名）
- 理疗师（2-3名）
- 药剂师（2名）

### 3. 保洁人员类
- 保洁人员（2名）

## 初始化步骤

### 方式一：在本地开发环境初始化

1. 进入backend目录：
```bash
cd backend
```

2. 运行初始化脚本：
```bash
python init_dys_jobs.py
```

3. 脚本会自动：
   - 清空现有职位数据
   - 添加9个德元升中医馆的职位
   - 按分类和优先级排序

### 方式二：在Ubuntu服务器初始化

1. 停止正在运行的服务（Ctrl+C）

2. 拉取最新代码：
```bash
cd /path/to/dys
git pull origin main
```

3. 运行初始化脚本：
```bash
cd backend
python init_dys_jobs.py
```

4. 重启服务：
```bash
cd ..
python main.py
```

## 数据库变更

新增字段：
- `category` - 职位分类（管理/技术人员/保洁人员）
- `priority` - 优先级（数字越小越靠前）
- `position_count` - 招聘人数

## 注意事项

⚠️ **重要**：运行初始化脚本会清空所有现有职位数据和相关申请记录！

如果需要保留现有数据，请在运行脚本前备份数据库：
```bash
cp recruitment.db recruitment.db.backup
```

## 自定义职位

如需修改职位信息，编辑 `backend/init_dys_jobs.py` 文件中的 `jobs_data` 列表。

重要字段说明：
- `category`: 必须是"管理"、"技术人员"或"保洁人员"之一
- `priority`: 数字越小越靠前（1-9）
- `position_count`: 招聘人数
- `status`: "active"（激活）、"closed"（关闭）、"draft"（草稿）
